## 네트워크 개념

- 통신망 종류
    - **LAN**
        - **Ethernet/ IEEE 802.3 LAN : 매체 제어 접근 방식 → CSMA/CD 사용**
        - Token Ring/ IEEE **802.5 LAN** : Token Passing 사용
        - FDD: 이중 링 구조, Token Passing 사용
    - **MAN**
        - **DQDB** : 이중버스 토폴로지 사용
    - **WAN**
        - 구성방식: 전용선,회선교환망,패킷교환망
        - 전용회선,회선교환망 → 프로토콜: HDLC,PPP
        - 패킷교환망 →**프로토콜: X.25 (패킷망에서 DCE(회선 종단장치)와 DTE(데이터 단말장치)사이에 이루어진 상호 작용을 위한 프로토콜)**
    - SAN
        - Storage Area Network
        - 호스트 컴퓨터의 종류에 구애받지 않고, 별도의 연결된 저장장치 사이에 대용량 데이터 전송 가능
        
- LAN 토폴로지
    - 성형(스타형),망형,버스형,링형,트리형
    - 설치비용이 많이 들지만, 장애 발생시 다른 시스템에 영향이 적고 신뢰성이 가장 높은 구성방식?
    **→ 망형**
    - 논리적 순환형으로, 하나의 노드장애가 전체 토폴로지에 영향을 미치는 방식은?
    **→ 링형 (+ 토큰 패싱방법)**

## 네트워크 장비

- LAN 구성 장비
    - 케이블 (전송거리 가능 순서 : CX → T,TX → FX)  *** LX : 5km 이상 가능**
    - 리피터: 증폭 기능
    - 허브: 네트워크 확장, 연결,증폭
    - 브릿지: 주소에 따라 **목적지 포트로 프레임 전달**
    - 스위치: 브릿지와 비슷, 더 빠르게 **MAC Address Table** 기반으로 프레임 전송

- 인터넷 네트워킹 장비
    - 라우터 : 
    - OSI 1-3 계층 기능 지원
    - LAN 연결, 경로 설정
    - **게이트웨이 :** 
    - 라우터와 거의 비슷
    - **서로 다른 형태의 네트워크를 상호 접속**

- UTP 케이블링
    - T568A: 녹 띠주 파 주 갈
    - T568B:  주 띠녹 파 녹 갈
    

## 프로토콜 개요와 기능

- OSI 7 모델 
: **물리 → 데이터링크(오류검출) → 네트워크(경로 관리) → 전송(종단 간 연결) → 세션 (종단 간 연결 설정,유지,해제)→ 표현 → 응용(UI 제공)**
    
    **Q. 데이터의 암호화와 해독 수행하고, 효율적인 전송을 위해 압축과 해제를 수행하는 계층?** → **표현 계층**
    
- 프로토콜
: **형식 + 의미 + 순서**
- 전송계층 프로토콜
    - TCP : **연결 설정 (3-Way handshacking) ,연결 해제(4-Way handshacking)**
        - 3 - Way : SYN → ACK/SYN → **ACK**
    - UDP : 비연결 지향, 멀티캐스트/브로드캐스트에 적합

- 응용 계층 프로토콜
    - POP : 전자우편 수신,보관
    - Telnet : CUI 기반 원격지 컴퓨터 접속 지원 인터넷 표준 프로토콜
    - SSH : 위의 보안기능 강화
    - SNMP:  네트워크 장비 관리 감시하여 특정 망의 상태 파악
    

### IP 주소와 도메인

- **IPV4 주소 체계**
    - **4개의 옥텟 → 총 32비트**
    - **네트워크 ID 와 호스트 ID**로 구성되어있음
        
        <aside>
        💡
        
        - **Limited BroadCast** 주소: 255.255.255.255
        - **Loopback 주소:** 127.0.0.0 ~ 127.255.255.255 → 네트워크 부분 값이 Class A에 속함
        - **디폴트 값**: 0.0.0.0
        - **사설 IP 주소**: 라우팅이 불가능한 주소, 인터넷에서 사용할 수 없음
        10.0.0.0 ~ 255
        172.16.0.0 ~ 172.31.255.255
        192.168.0.0 ~ 192.168.255.255
        </aside>
        
    - **서브넷 마스크**: 네트워크 부분과 호스트 부분을 구분해 주는 값
        - ex) **netmask 값이 255.255.255.128 일 경우,**
        128 을 살펴보기 
        1. 2진수 8자리 숫자로 128을 만드려면, **10000000 이다. 한 비트만 사용**이 되었다.
        2. 이때 분할되는 서브넷 개수는 2^1 = 2개이다.
        3. **서브넷당 전체 호스트 개수는 안쓴 비트 개수 >> 7개 >> 2^7 - 2 = 126개**이다.
        4.  **사용 가능한 총 IP 주소 개수는 126개 * 2 = 252개**
